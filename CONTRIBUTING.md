
# Contributing Guide

  

## 📌 Branching Strategy

  

- main: production-ready code

- develop: integration branch

- feature/*: new features

- fix/*: bug fixes

---  

## 🚀 Workflow

1. Pull latest code from develop

2. Create a new branch

3. Code & commit

4. Push branch

5. Create Pull Request

6. Code review

7. Merge

  

---

  

## 🧾 Commit Convention

  

- feat: new feature

- fix: bug fix

- refactor: improve code

- docs: documentation

- chore: setup/config

  

---

  

## 🔍 Code Review Rules

  

- At least 1 approval required

- No self-merge

- Must pass basic testing

---

## ⚠️ Rules

- Do NOT push directly to main or develop

- Always create PR

- Always pull before coding
